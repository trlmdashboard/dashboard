Runs entirely in your browser — no data leaves this page. Because most third-party sites don't send CORS headers, checks use a "no-cors" probe: the browser attempts the connection but can't read the response body or status code, only whether the request completed. This is a solid signal for "is something answering at this address" but can occasionally miss server-side errors on same-origin-friendly sites. Each tracker keeps its own independent history for this session only, and everything resets on page reload.
